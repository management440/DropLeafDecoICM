# DropLeaf Deco — Inventory Command Center (ICM)

A Streamlit-based operations dashboard for managing, reviewing, and exporting vintage furniture inventory via Supabase.

---

## Project Structure

```
C:\icm_project\
│
├── 01_ingest/
│   ├── manifests/        # Drop CSV manifests here before ingestion
│   ├── assets/           # Drop SKU-matched images here before ingestion
│   └── archive/          # Processed manifests are moved here automatically
│
├── 02_export/            # Timestamped CSVs generated by the export pipeline
├── 03_archive/           # CSVs moved here after automated git push
│
├── modules/
│   └── marketplace_integration/
│       ├── export_module.py      # Queries Supabase, writes CSV
│       ├── automation_module.py  # Orchestrates export → archive → git push
│       └── dashboard_bridge.py   # Streamlit button wired to the pipeline
│
├── dashboard.py          # Main Streamlit app
├── ingest_manifest.py    # Ingests CSV manifests into Supabase
├── ingest_assets.py      # Uploads SKU images
├── export_marketplace.py # Standalone export script (legacy)
├── validate_manifest.py  # Pre-ingest manifest validation
└── research_module.py    # Tavily web research integration
```

## Setup

**Prerequisites:** Python 3.11+, a Supabase project, a Tavily API key.

```powershell
cd C:\icm_project
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

Create a `.env` file in the project root (never commit this):

```
SUPABASE_URL=your_supabase_url
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
TAVILY_API_KEY=your_tavily_key
```

## Running the Dashboard

```powershell
.\venv\Scripts\activate
streamlit run dashboard.py
```

Open `http://localhost:8501` in your browser.

## Workflow

### 1. Ingest
Drop a CSV manifest into `01_ingest/manifests/` with columns matching the `items` table schema, then run:
```powershell
python ingest_manifest.py
```
Drop SKU-matched images into `01_ingest/assets/` and run:
```powershell
python ingest_assets.py
```

### 2. Manual Review
Items flagged `manual_review_required` appear in the **Manual Review Queue** tab. Use the web research tool to look up provenance, verify the title, and commit the item to inventory.

### 3. Export
In the **Marketplace Export** tab, click **Export Ready Items to CSV**. This triggers the full automation pipeline:
1. Queries Supabase for all items with `status = 'ready_for_export'`
2. Writes a timestamped CSV to `02_export/`
3. Moves the CSV to `03_archive/`
4. Stages only the export and archive directories (guard rejects unexpected files)
5. Commits with message `Automated archive: [timestamp]`
6. Pushes to `origin/main`

## Troubleshooting

| Symptom | Fix |
|---|---|
| `SUPABASE_URL` KeyError | `.env` file missing or not in project root |
| Export button shows "No items ready" | No rows with `status = 'ready_for_export'` in Supabase |
| Git push fails in automation | Check remote auth; ensure SSH key or HTTPS credential is configured |
| Automation guard aborts with unexpected files | Stage and commit those files manually before running the export |
| Dashboard shows stale module after code change | Restart Streamlit — hot-reload doesn't refresh imported modules |
